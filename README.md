# 🚀 Rubika Cloud Leecher (GitHub Actions Edition)

این پروژه به شما امکان می‌دهد فایل‌های حجیم (تا ۵ گیگابایت و بیشتر) را مستقیماً از لینک مستقیم به **پیام‌های ذخیره شده (Saved Messages) روبیکا** منتقل کنید. این کار با استفاده از سرورهای پرقدرت گیت‌هاب (GitHub Actions) انجام می‌شود و هیچ حجمی از اینترنت شما مصرف نمی‌کند.

### ✨ ویژگی‌ها:
*   **سرعت دانلود فوق‌العاده:** استفاده از `aria2c` با ۱۶ اتصال همزمان.
*   **دور زدن محدودیت حجم:** تقسیم خودکار فایل‌ها به پارت‌های ۲ گیگابایتی (سقف مجاز روبیکا).
*   **آپلود موازی:** قابلیت آپلود همزمان ۳ پارت برای افزایش سرعت.
*   **امنیت سشن:** استفاده از GitHub Secrets برای جلوگیری از لو رفتن اکانت.

## 🛠 مراحل راه‌اندازی

### مرحله ۱: ساخت فایل سشن (Session) در سیستم خودتان
روبیکا برای ورود نیاز به کد تایید دارد. چون گیت‌هاب محیط گرافیکی ندارد، ابتدا باید سشن را در سیستم خود بسازید:

1.  پایتون را نصب کنید.
2.  یک پوشه بسازید و ترمینال را در آن باز کنید.
3.  یک محیط مجازی (Virtual Env) ساخته و کتابخانه را نصب کنید:
    ```bash
    python -m venv venv
    source venv/bin/activate  # در ویندوز: venv\Scripts\activate
    pip install rubpy
    ```
4.  فایل `session_generator.py` (موجود در این ریپازیتوری) را اجرا کنید و شماره موبایل و کد تایید را وارد کنید.
5.  خروجی: یک کد طولانی در ترمینال چاپ می‌شود. آن را کپی کنید و در جایی امن نگه دارید.
   
### مرحله ۲: تنظیمات در GitHub
1.  به مخزن خود در گیت‌هاب بروید.
2.  وارد بخش **Settings > Secrets and variables > Actions** شوید.
3.  یک `New repository secret` بسازید:
    *   **Name:** `SESSION_FILE_BASE64`
    *   **Value:** (آن کد طولانی که در مرحله قبل به دست آوردید را اینجا پیست کنید).

### مرحله ۳: آماده‌سازی فایل‌های مخزن
مطمئن شوید فایل‌های زیر در ریشه (Root) مخزن شما وجود دارند:
1.  **`rubika_uploader.py`**: اسکریپت اصلی آپلودر.
2.  **`.github/workflows/smart-downloader.yml`**: فایل پیکربندی گیت‌هاب برای اجرای عملیات

### مرحله 4: 🚀 نحوه اجرا و استفاده

روند کار بسیار ساده است و مشابه قبل انجام می‌شود:

1.  آدرس فایل مورد نظرتان را در برنامه **OctoFetch** وارد کنید تا اکشنِ گیت‌هاب استارت بخورد. در این مرحله دو حالت پیش می‌آید:

#### **حالت اول: فایل‌های کم‌حجم**
اگر فایل کوچک باشد، عملیات سریع انجام شده و یک پیغام سبز رنگ با متن **"Action completed successfully"** دریافت می‌کنید. 
*   **نکته:** احتمالاً بلافاصله بعد از این پیام، یک ارور مشاهده می‌کنید؛ این ارور اصلاً مهم نیست و آن را نادیده بگیرید. (دلیل آن تغییرات در فایل `smart-downloader.yml` است که با ساختار اصلی برنامه OctoFetch کمی ناهماهنگ شده، اما در روند آپلود اختلالی ایجاد نمی‌کند).

#### **حالت دوم: فایل‌های سنگین و حجیم**
اگر فایلی که لینک دادید سنگین باشد، به دلیل زمان‌بر بودن فرآیند دانلود، تقسیم‌بندی و آپلود، ممکن است برنامه OctoFetch بعد از مدتی ارور بدهد یا متوقف شود.
*   **اصلاً نگران نباشید!** در این حالت، اگرچه برنامه متوقف شده، اما در پشت صحنه **سرورهای گیت‌هاب هنوز مشغول کار هستند.**
*   کافیست به تب **Actions** در پروژه گیت‌هاب خود بروید. آنجا می‌بینید که عملیات همچنان در حال اجراست و پارت‌های فایل شما یکی‌یکی در حال آپلود شدن به روبیکا هستند.

#### **نتیجه نهایی:**
وقتی در پنل گیت‌هاب، اکشن مورد نظر به اتمام رسید و تیک سبز خورد، فایل شما (یا پارت‌های آن) در بخش **Saved Messages (پیام‌های ذخیره شده)** روبیکا در دسترس خواهد بود.

> **نکته پایانی:** (البته اگه خدا بخواد و روبیکا وسط کار اتصال رو قطع نکنه، گیت‌هاب اکانت رو نبنده، بهمون بمب نزنن و هزار تا دلیل دیگه که ممکنه پیش بیاد... خلاصه که موفق باشید! 😉)

## 📦 نحوه چسباندن پارت‌ها به هم
پس از آپلود تمام پارت‌ها در روبیکا، آن‌ها را دانلود کرده داخل یک پوشه بریزید و با روش‌های زیر به هم متصل کنید (فقط یادتون باشه اسم فایل رو هرچی خاستید بزارید ولی پسوند رو باید پسوند فایل اصلی بزارید که میخواستید دانلود کنید مثلا mp4/zip/mkv/...):

*   **ویندوز (CMD):**
    ```cmd
    copy /b part_* full_file.zip
    ```
*   **اندروید:** از برنامه **ZArchiver** استفاده کنید (پارت‌ها را انتخاب و Extract کنید) (اندروید تست نکردم ولی میگن میشه) .
*   **لینوکس:**
    ```bash
    cat part_* > full_file.zip
    ```
